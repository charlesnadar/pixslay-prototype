## 1. The Problem

**What is broken:** Not taking photos (solved). Not storing them (largely solved). The broken step is: sharing the right photos with the right people, automatically, at full quality, without anyone doing extra work.

### Why each platform fails

**WhatsApp:** 60–70% quality loss. Ask problem ("send me the solo ones"). Storage wars. No organisation. Photos drown in chat within hours.

**Google Drive:** Folder naming hell. Access friction ("you need to request access"). Nobody opens it. Broken links.

**Google Photos Shared Albums:** Requires deliberate setup. Android/iOS wall. Face grouping is private — cannot tag friend's face. Notification blindness.

**AirDrop / iCloud:** iOS only. Requires physical proximity. iCloud Shared Albums almost unknown.

**Root cause across all:** Every platform treats photos as files to transfer — not memories to deliver to the right people. The photographer does all the work. The person in the photo has no way to find themselves.

Splitwise did not invent expense splitting. PixSlay is doing the same for group photo delivery — building the infrastructure layer for something everyone already does.

Three enablers now that weren't achievable five years ago:

## 3. The Product

**PixSlay is a group memory platform** — not a trip planner, not a photo editor, not a social feed. A memory wallet for groups where photos arrive automatically, are organised intelligently, and are accessible forever.

### The Four Tabs

| Tab | What Lives Here | Free or Paid |
|-----|----------------|-------------|
| 📸 Gatherings | Create group, sync photos, face + location tagging, album listing, likes + comments, HD download, highlight reel, memories | Viewing free. Contributing = subscription. |
| 💬 Planning | Packing list, document vault, carpool, flight + train live status, group arrival timeline, safety check-in | Free always |
| 💰 Expenses | View + log expenses, who owes whom, UPI settlement (Phase 3) | Viewing + logging free. Settlement Phase 3. |
| 👤 Profile | Travel map, group identity card, friend overlap map, gathering history, subscription settings | Basic free. Map + identity card = subscription. |

### The Two Core Users

| Role | Who They Are | Their Journey |
|------|-------------|--------------|
| The Organiser | Plans every trip, main photographer. Already a subscriber. | Creates group → syncs photos automatically → photos delivered to all members tagged to right faces. |
| The Returning Member | Was on previous trips with this group. | Opens notification. All photos waiting — already identified, already tagged. Zero friction. Pure magic. |

### The Three Identification Moments

- **Moment A — Which one is you?** Grid of detected faces. User taps themselves. Known globally from this point.
- **Moment B — Who is this?** Unidentified faces named one by one. Suggestion chips from member list.
- **Moment C — Select and share.** Long press → selection mode → share to everyone, specific people, or download.

## 7. Business Model

### The Global Permission Logic
> Consuming is always free. Contributing requires subscription. Planning and coordination are always free. Safety is always free. AI generation is funded by the subscriber and shared free with the whole group.

### Subscription Plans (all GST-inclusive)

| Plan | Price | Duration | Best For |
|------|-------|----------|---------|
| Free | ₹0 | Forever | Anyone joining or receiving photos |
| Fortnight | ₹49 | 15 days | One gathering, no commitment |
| Monthly | ₹79 | 30 days | Regular social person, 2+ gatherings/month |
| Annual | ₹599 | 365 days | Power user. Saves 35% vs monthly. |
| Organiser ≤20 | ₹249 | Gathering + 7 days | Small group, trek leaders |
| Organiser 21–50 | ₹399 | Gathering + 7 days | Medium group, wedding travel |
| Organiser 51–100 | ₹799 | Gathering + 7 days | Corporate offsites, medium events |
| Organiser 100+ | ₹1,499 | Gathering + 7 days | Large events, weddings |

**First organiser gathering free** (up to 50 participants). One per verified phone number, ever.

### Trial Mechanic
Trial does NOT start on install. Starts on first value creation — whichever comes first:
- First time they sync own photos to a group, OR
- First time they download an HD photo

**Trial duration:** 30 days. If gathering is ongoing when trial expires → extends automatically to 7 days after gathering end date.

## 12. Technology Architecture

### Stack

| Layer | Technology | Rationale |
|-------|-----------|----------|
| Mobile | React Native | One codebase iOS + Android. Largest India talent pool. |
| Backend | Node.js (Phase 1–2) → Python FastAPI microservice (Phase 3) | Speed now. Correctness later. Avoid EVJoints migration mistake. |
| DB (prototype/beta) | Firebase Firestore | Free tier, fast setup, no DevOps. Sufficient to 5,000 users. |
| DB (production) | Supabase (PostgreSQL) | Relational for face graph + gathering queries. Migrate at Series A or 10K users. |
| Photo/video storage | GCS Mumbai region (mandatory) | Lifecycle: Standard → Nearline → Coldline → Archive. Photos forever. DPDP Act. |
| CDN | Cloudflare in front of GCS | Zero egress fees. Free tier covers Phase 1 + 2 entirely. |
| Video | FFmpeg — 720p compression on upload | 4x storage cost reduction. Original stays on device. |
| Push notifications | Firebase Cloud Messaging | Free. Unlimited. iOS + Android. |
| Payments | Razorpay | UPI, cards, net banking. 2% per transaction. |
| Face + location AI | Google Vision API — organiser photos only | 1,000/month free. ₹125 per 1,000 after. Hard spend cap in Google Cloud. |
| Payments | Razorpay | UPI, cards, net banking. 2% per transaction. |
| Flight status | AviationStack API | Free tier 500 calls/month. |
| Train PNR | RapidAPI — Indian Railways wrapper | ₹800/month. Avoid IRCTC official API Phase 1. |
| Highlight reel | Shotstack API | ₹20 per reel. Phase 2 only. |

### Non-Negotiable Architecture Decisions

| Decision | Why It Cannot Change Later |
|---------|--------------------------|
| Photos stored in group object (not user account) | Cancellation cannot cascade into group data loss |
| Signed URLs — 15-minute expiry | Permanent URLs cannot be revoked — privacy permanently broken |
| GCS Mumbai region from day one | DPDP Act — face biometric = sensitive personal data |
| Vision API on organiser photos only | Affects per-gathering cost model. Change breaks organiser pricing. |
| UPI ID in user profile from day one | Adding fields to production records at scale is risky |
| Hard Google Cloud spend cap | Viral moment without cap = cost spike that bankrupts bootstrapped product |
| Gathering end date at creation | Cannot be inferred retroactively from photo timestamps |

## 13. Security and Privacy (DPDP Act 2023)

- Explicit consent before collecting biometric data — dedicated consent screen before Moment A. Two clear buttons. Cannot be in T&C.
- Data minimisation — delete raw face embeddings after identity confirmed
- Right to erasure — architecture must support from day one
- GCS Mumbai region mandatory from first user
- Privacy policy + T&C required before first user (budget ₹8,000 for legal)
- Signed URLs only, 15-minute expiry — every request checks group membership
- API rate limiting on all endpoints
